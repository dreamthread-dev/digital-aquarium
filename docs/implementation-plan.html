PNG Base64 → Texture2D 変換

# WebSocketClient.gd
func create_texture_from_base64(data_url: String) -> ImageTexture:
    # "data:image/png;base64," プレフィックスを除去
    var base64_str = data_url.split(",")[1]
    var bytes = Marshalls.base64_to_raw(base64_str)
    var image = Image.new()
    image.load_png_from_buffer(bytes)
    return ImageTexture.create_from_image(image)

接続管理の設計

# 接続種別で管理を分離
tablet_connections: set[WebSocket] = set()  # 送信元
godot_connections:  set[WebSocket] = set()  # 受信先
fish_queue: asyncio.Queue = asyncio.Queue(maxsize=50)

# type フィールドで接続種別を判定
if data["type"] == "godot":
    godot_connections.add(websocket)
elif data["type"] == "fish":
    await fish_queue.put(data)   # キューに積む

放流キュー（0.5秒間隔）

async def queue_worker():
    while True:
        fish_data = await fish_queue.get()
        for ws in list(godot_connections):
            try:
                await ws.send_json(fish_data)
            except:
                godot_connections.discard(ws)
        await asyncio.sleep(0.5)   # 0.5秒間隔
⚠️ 1024×1024 PNG の Base64 は約 1.4MB。LAN 内（5GHz）なら問題ないが、キューの maxsize=50 で過負荷を防ぐ。

Flood Fill アルゴリズム

// 完全一致 Flood Fill（二値化済み線画前提）
function floodFill(ctx, x, y, fillColor) {
  const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
  const data = imageData.data;
  const targetColor = getPixelColor(data, x, y);

  // 黒ピクセル（境界線）またはすでに塗られていたら終了
  if (isBlack(targetColor) || isSameColor(targetColor, fillColor)) return;

  // スタックベース BFS で塗りつぶし（再帰を避けてスタックオーバーフロー防止）
  const stack = [[x, y]];
  while (stack.length > 0) {
    const [cx, cy] = stack.pop();
    if (!isSameColor(getPixelColor(data, cx, cy), targetColor)) continue;
    setPixelColor(data, cx, cy, fillColor);
    stack.push([cx+1,cy],[cx-1,cy],[cx,cy+1],[cx,cy-1]);
  }
  ctx.putImageData(imageData, 0, 0);
}
💡 線画は事前に Python（Pillow）で二値化（白 or 黒のみ）しておくことで、アンチエイリアスによる漏れが発生せず、完全一致 Flood Fill が確実に動作する。

🚀 起動・統合フロー

起動順序

1
PC の固定 IP を確認 / ホットスポットを起動

ipconfig で IP アドレスを確認。タブレットの WebSocket URL に使用。

2
Python サーバーを起動

uvicorn main:app --host 0.0.0.0 --port 8000

3
Godot アプリを起動

WebSocket 接続確認。デバッグコンソールに「Connected to server」が表示されれば OK。

4
タブレットで http://{PC_IP}:8000 を開く

Chrome で URL を入力。接続インジケーターが緑になることを確認。

5
塗り絵 → 放流 → 巨大スクリーンに魚出現を確認

E2E の動作確認。F1 でデバッグ UI を開き Boids パラメータを調整。

Windows Firewall 設定

# 管理者権限の PowerShell で実行
New-NetFirewallRule -DisplayName "Digital Aquarium" `
  -Direction Inbound -Protocol TCP -LocalPort 8000 -Action Allow
